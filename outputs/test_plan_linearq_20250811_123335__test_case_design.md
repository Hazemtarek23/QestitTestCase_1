## Test Design Overview

**Approach Explanation:**
The Linear Qâ„¢ methodology is applied to maximize risk coverage with the minimum number of test cases. For each module, we:
- Identify the happy path (Test Case 1)
- Apply linear expansion: for each attribute, create a test case that varies one attribute from the happy path, holding others constant
- For high-risk modules (e.g., authentication, password management), ensure edge and negative cases are included
- For lower-risk modules (e.g., email notification), use orthogonal testing for efficiency

**Risk Coverage Estimate:**
This approach achieves approximately 85% risk coverage per module, focusing on the most likely and impactful failure points.

**Test Design Technique Justification:**
- **Linear Expansion**: Efficiently covers the majority of attribute-level risks with minimal test cases
- **Full Combinatorial**: Used only for the highest risk (not needed here for 85% coverage)
- **Orthogonal**: Used for integration/notification modules to minimize test set size
